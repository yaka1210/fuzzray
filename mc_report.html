Оценка эксплуатируемости

Оценка показывает, насколько обнаруженная уязвимость близка к практической эксплуатации (т.е. может ли злоумышленник использовать её для выполнения произвольного кода, повышения привилегий или утечки данных). Оценка формируется на основе балльной системы из трёх компонентов: тип уязвимости (CWE), таксономия поведения (место сбоя, область памяти, состояние потока управления) и данные отладчика (адрес сбоя, тип инструкции).

УровеньОписаниеПримеры
Эксплуатируемая Прямой контроль потока выполнения: указатель инструкций ($pc) повреждён или переход по контролируемому адресу Возврат на невалидный адрес, повреждённый косвенный вызов (vtable/GOT), $pc в невыделенной памяти
Вероятно эксплуатируемая Запись в память или повреждение heap-метаданных без прямого контроля $pc Out-of-bounds write (CWE-787), Use-after-free (CWE-416), Double-free (CWE-415), Format string (CWE-134)
Вероятно не эксплуатируемая Чтение за пределами буфера, детерминированный NULL deref или арифметическая ошибка — обычно приводят к DoS, но не RCE Out-of-bounds read (CWE-125), NULL dereference (CWE-476), Division by zero (CWE-369), Integer overflow (CWE-190)
Не определено Недостаточно данных для оценки — тип уязвимости не определён или отсутствует информация от отладчика UBSan trap без маппинга, crash без backtrace, неизвестный сигнал
